\section{Identity and scope}
\begin{itemize}
  \item \textbf{Name and version}: MLR-X (version 1.0).
  \item \textbf{License}: GNU Affero General Public License v3.0 (AGPL-3.0) with 2025 rights held by Jackson J. Alcázar. Windows build metadata mirrors this information.
  \item \textbf{Audience}: users who need to fit, validate, and diagnose multiple linear regression models on small or large datasets.
  \item \textbf{Official channels}:\\
  \textbf{Repository}: \url{https://github.com/Jacksonalcazar/MLR-X}\\
  \textbf{Homepage}: \url{jacksonalcazar.github.io/MLR-X}
\end{itemize}

\subsection{Distributions and compatibility}
\begin{itemize}
  \item \textbf{Available builds}: self-contained binaries for Linux, macOS (GUI), Windows GUI, and Windows GUI+CLI.
  \item \textbf{System requirements}: 64-bit Windows, Linux, or macOS. No Python installation or scientific libraries are required; everything ships within the build.
  \item \textbf{Console version}: available in the CLI-enabled builds (Linux and Windows GUI+CLI) to run the full pipeline without the GUI by supplying a \texttt{config.conf} file.
\end{itemize}

\section{Requirements and installation}
The distributed builds are self-contained downloads and do not require installers or shortcut generators. Place each executable in your preferred folder. Optionally, add that folder to your system \texttt{PATH} so you can launch \verb|MLR-X| from any directory (e.g., \verb|export PATH=/opt/MLRX:$PATH| on Linux or configuring environment variables on Windows). Choose the appropriate build and follow these steps:

\subsection{Setting up the portable builds}
\begin{itemize}
  \item \textbf{Linux}: download the binary, move it to a stable folder, and mark it as executable if needed (\texttt{chmod +x MLR-X}). To add it to your \texttt{PATH}, prepend the folder with \verb|export PATH=/path/to/folder:$PATH| or the equivalent shell profile setting. To register a desktop launcher:
        \begin{enumerate}[label=\arabic*.]
          \item Build the Linux executable (see the steps above), then copy the binary and the icon to a stable location, for example:
          \item \texttt{sudo mkdir -p /opt/MLRX}
          \item \texttt{sudo cp MLRX /opt/MLRX/}\\
                \texttt{sudo cp icon/icon.ico /opt/MLRX/}
          \item Update the \texttt{Exec} and \texttt{Icon} paths in \texttt{MLRX.desktop} if you choose a different installation directory.
          \item Install the launcher by copying it to your applications directory and mark the file as executable:
          \item \texttt{chmod +x MLRX.desktop}
          \item \texttt{cp MLRX.desktop ~/.local/share/applications/}
          \item \# or for all users:\\
                \texttt{\# sudo cp MLRX.desktop /usr/share/applications/}
          \item Refresh the desktop launcher database (optional but recommended):
          \item \texttt{update-desktop-database ~/.local/share/applications}\\
                \texttt{\# or sudo update-desktop-database /usr/share/applications}
        \end{enumerate}
  \item \textbf{Windows GUI}: download the GUI build, place \texttt{MLR-X.exe} in a convenient folder, and run it directly.
  \item \textbf{Windows GUI+CLI}: same as Windows GUI, with the Run CLI option enabled.
  \item \textbf{macOS GUI}: download the app bundle and run it from your preferred location (e.g., move it to \texttt{/Applications} if desired).
\end{itemize}

% =========================================================
\section{Command-line (CLI) execution}
Available only in the builds that include the CLI (Linux and Windows GUI+CLI). Run the pipeline headlessly with a configuration file:
\begin{lstlisting}[style=mlrx]
./MLRX  config.conf or ./MLRX  path/to/config.conf      # Linux
MLRX.exe config.conf or MLRX.exe  path/to/config.conf    # Windows GUI+CLI
\end{lstlisting}
Executing with a configuration file loads the defined parameters and runs the full pipeline in the console.

\section{Interface tour (GUI)}
MLR-X organizes its features into tabs within a main window.

\subsection{\textbf{Setup \& Run} tab}
\paragraph{Dataset}
\begin{enumerate}[label=\arabic*.]
  \item Select the CSV (or use \emph{Browse}).
  \item Define the delimiter (\verb|;| by default), the dependent variable, and non-predictor columns.
  \item Exclude observations by ID or range (e.g., \texttt{3,9,4-8}).
  \item Enable \emph{Exclude nearly constant predictors} and set the percentage threshold.
  \item Actions: \emph{Preview data}, \emph{Run analysis}, \emph{Cancel}. Status appears in the top-right corner of the frame; the task name appears in the bottom-right corner.
\end{enumerate}

\paragraph{Data splitting}
\begin{itemize}
  \item \textbf{None}: use the full dataset.
  \item \textbf{Random}: specify \emph{Test size (\%)}.
  \item \textbf{Manual}: lists or ranges for train/test without overlap.
  \item \textbf{External}: holdout file with the same target and columns.
\end{itemize}

\paragraph{Settings}
\begin{enumerate}[label=\arabic*.]
  \item Search method: \textbf{EPR-S} (Expand--Perturb--Reduce--Swap) or \textbf{All Subsets}; contextual help (\emph{ℹ} icon).
  \item Key parameters: \emph{Max predictors}, \emph{Seeds}, \emph{Seed size}, significance levels, \emph{export limit}, and parallel jobs (\texttt{n\_jobs}).
  \item Target metric (R$^2$, adj-R$^2$, or Q$^2$), covariance type (\texttt{nonrobust}, \texttt{HC0--HC3}), prediction clipping, and recommendation buttons.
  \item Interactive validation for numeric fields, paths, and clipping thresholds.
\end{enumerate}

\subsection{\textbf{Models} tab}
Three synchronized tables plus an observation diagnostic view. Selecting a model enables the \textbf{Validation}, \textbf{Diagnostics}, \textbf{Visualization}, and \textbf{Summary} tabs.
\begin{itemize}
  \item \textbf{Top models (train)}: ID, predictor list, R$^2$/adj-R$^2$, RMSE, MAE, $s$, maximum and average VIF.
  \item \textbf{Internal validation}: Q$^2$/RMSE/MAE/$s$ for LOO and k-fold.
  \item \textbf{External validation}: Q$^2_{\mathrm{F1,F2,F3}}$, external RMSE, and MAE.
  \item \textbf{Observation diagnostics}: observed/predicted values (standard and LOO), residuals, $z$, hat, standardized residuals, and Cook's distance.
  \item Controls: \emph{Sorted by}, \emph{Max rows} (up to 5000), and synchronized refresh with other tabs.
\end{itemize}

\subsection{\textbf{Validation} tab}
\begin{itemize}
  \item Model selection: \emph{Top}, \emph{Multiple} (range like \texttt{1-5,7,9}), or \emph{All}.
  \item \textbf{LOO and k-fold}: internal validation with \emph{Folds} and \emph{Repeats}; plausibility rules (folds $\ge 2$, repeats $\ge 1$, folds $\le n$).
  \item \textbf{External validation}: select file and delimiter; requires the same target column as the training set.
  \item Independent status indicators for each process and a final summary in the log.
\end{itemize}

\subsection{\textbf{Diagnostics} tab}
\begin{itemize}
  \item Dataset filter (Training/Testing/Both) with holdout status propagated.
  \item Key columns: observation, set, actual and predicted values (standard and LOO), residuals, $z$, hat, standardized residuals, and Cook's distances (standard and LOO).
  \item \emph{Export CSV} is available when data are visible; it respects the active filter.
  \item Correlation matrix (lower triangle) with an option to include the dependent variable.
\end{itemize}

\subsection{\textbf{Visualization} tab}
\begin{itemize}
  \item Available plots: heatmaps (with or without dependent variable), Predicted vs Actual (standard and LOO), residuals vs actual/predictions, Scale--Location, QQ residual, residual distribution, Williams plot, Cook vs leverage, Hat-values, and Predicted vs Hat.
  \item Common controls: legend toggle and position, labeling mode, identity line, $h^*$ color, optional linear fit, grid, distinct styles for train/test, synchronized ranges, and export to PNG/TIFF/PDF/SVG.
  \item Axis labels and font size can be edited directly from the interface.
\end{itemize}

\subsection{\textbf{Summary} tab}
\begin{itemize}
  \item Re-fits OLS when switching models and reports the recommended covariance type (HC3 by default when heteroscedasticity or high leverage is detected).
  \item Metrics: R$^2$/adj-R$^2$, RMSE, MAE, $s$, VIF, correlations, advanced statistics (F, log-likelihood, AIC/BIC, Omnibus/JB, skewness, kurtosis, Durbin--Watson, condition number, Lin's concordance coefficient).
  \item \emph{Export Summary...} saves the complete text dossier.
\end{itemize}

\subsection{\textbf{Desktop} tab (Variable Explorer)}
\begin{itemize}
  \item Load CSV data or previous results (results enable extra functions).
  \item Variable scope: \emph{All variables} or \emph{Only variables of model} (enter an ID).
  \item Appearance controls: marker size/color/style, axis limits and steps, aspect ratio (1, 4:3, 16:9, 9:16).
  \item Interactive labeling (left-click to activate/drag, right-click to clear), legend, grid, linear fit, and \emph{Save current plot}.
\end{itemize}

\subsection{\textbf{Help} tab}
Displays the built-in quick help with license notice, project links, and the embedded shortcut to the issue tracker.

\section{Analysis flow and process handling}
\subsection{Before starting}
\begin{enumerate}
  \item Load and preview the dataset.
  \item Review delimiter, target variable, exclusions, and split mode.
  \item Confirm the export path (\texttt{models.csv} by default).
\end{enumerate}

\subsection{Execution}
\begin{enumerate}
  \item Ensure no other run is active.
  \item Click \emph{Run analysis}; the app validates paths, limits, and reloads the dataset.
  \item Status switches to \emph{Running}, previous results are cleared, and the real-time log starts.
\end{enumerate}

\subsection{Progress and cancellation}
\begin{itemize}
  \item Bottom progress bar (\%) and \emph{Log} show evaluated models and events.
  \item \emph{Cancel} switches to \emph{Cancelling...} and stops jobs with orderly shutdown.
  \item Errors raise a dialog and set status to \emph{Error}.
\end{itemize}

\subsection{Results and metadata}
\begin{enumerate}
  \item The CSV is written to the confirmed path (\texttt{models.csv} by default).
  \item The first line contains a JSON block between \verb|#EPRS-S ... ;#| with run metadata.
  \item The last export path is remembered as the default for future sessions.
\end{enumerate}

\section{Search algorithms}
\subsection{EPR-S (Expand--Perturb--Reduce--Swap)}
\begin{enumerate}
  \item Define a target metric and an initial set with a predictor limit.
  \item \textbf{Expand}: add predictors that improve the metric without exceeding the maximum.
  \item \textbf{Perturb}: 1--1 swaps to escape local optima.
  \item \textbf{Reduce}: remove predictors when quality holds or improves.
  \item \textbf{Correlation cleanup}: if $|r|$ exceeds the threshold, keep the most useful variable.
  \item \textbf{Swap}: controlled reintroductions when they provide gains; convergence is reached when no gains remain.
  \item Quality filters: VIF threshold and \emph{Minimum report R$^2$/Q$^2$}; keeps unique models sorted.
\end{enumerate}

\subsection{All Subsets}
Generates and evaluates all combinations up to \emph{Max predictors}; recommended when there are $\lesssim 25$ predictors. Shares VIF and threshold filters with EPR-S.

\subsection{Engine good practices}
\begin{itemize}
  \item Seeds: at least as many as predictors; use the \emph{Recommended} button.
  \item Seed size: about half of \emph{Max predictors}.
  \item VIF threshold: keep it $<5$.
  \item Correlation threshold: between 0 and 1; suggested significance level 0.05.
\end{itemize}

% =========================================================
\section{Result format and exports}
\subsection{Model CSV}
\begin{itemize}
  \item Includes training metrics, internal validation (LOO/k-fold) and external validation, plus predictor counts and VIF.
  \item Metadata in the first line (JSON block) with process info, CPU time, and the number of models found and explored.
  \item \emph{export\_limit} controls the maximum rows exported.
\end{itemize}

\subsection{Summary and reports}
\begin{itemize}
  \item \emph{Export Summary...}: saves the textual dossier for the selected model.
  \item \emph{Export CSV} in Diagnostics: respects filters and the active dataset.
  \item Plots: \emph{Save current plot} exports to PNG/TIFF/PDF/SVG with auto-elevated DPI (\ensuremath{\ge}1000 if the current one is lower).
\end{itemize}

\section{Command-line usage (CLI)}
\subsection{Configuration file structure}
Text file with headed blocks (\# Dataset, \# Method, \# Validation, \# Output, etc.) and key--value pairs. List options allow index selection via \texttt{selected option: <n>} inside the block.

\paragraph{Main fields}
\begin{longtable}{>{\bfseries}p{4.2cm}p{10.2cm}}
Data path & Path to the input CSV.\\
Delimiter & Separator (\verb|;| by default, \verb|,|, tab, or literal text).\\
Dependent choice & Target variable (\verb|last| or column name).\\
Non variable spec & Columns to exclude from the predictor space.\\
Exclude constant & Enables removal of nearly constant columns (\emph{constant\_threshold} in \%).\\
Excluded observations & IDs or ranges to omit.\\
Max vars / n seeds / seed size & EPR-S engine parameters.\\
Cov type & \texttt{nonrobust}, \texttt{HC0}, \texttt{HC1}, \texttt{HC2}, \texttt{HC3}.\\
Signif lvl & Significance level for filters.\\
Corr threshold / VIF threshold & Limits to detect collinearity.\\
Report R2 threshold & Minimum threshold to report models.\\
Export limit & Maximum number of models in the CSV.\\
n jobs & Cores to use (normalized to at least 1).\\
Iterations mode & \texttt{auto}, \texttt{manual} (requires \emph{max\_iterations\_per\_seed}) or \texttt{converge}.\\
Clip predictions & Enable clipping with \emph{clip\_low}/\emph{clip\_high}.\\
Split mode & \texttt{none}, \texttt{random} (\% test), \texttt{manual} (train/test IDs) or \texttt{external} (path and delimiter).\\
Output path & Result CSV path.\\
Target metric & Objective metric (R$^2$ and available variants).\\
Method & \texttt{eprs} or \texttt{all\_subsets}.\\
\end{longtable}

\paragraph{Minimal example}
\begin{lstlisting}[style=mlrx]
# Dataset
data_path: data.csv
delimiter: ;
dependent_choice: last
non_variable_spec: ID
exclude_constant: true
constant_threshold: 90

# Method
method: eprs
max_vars: 10
n_seeds: 20
seed_size: 4
cov_type: HC3
signif_lvl: 0.05
corr_threshold: 0.90
vif_threshold: 4.0
report_r2_threshold: 0.80
iterations_mode: auto
export_limit: 500
n_jobs: 4

# Validation
split_mode: random
random_test_size_percent: 20

# Output
output_path: results/models.csv
target_metric: R2
clip_enabled: false
\end{lstlisting}

\subsection{Execution and progress}
After loading the file, the CLI prints dataset basics, selected options, and validates numeric ranges. During the search it reports stage progress (seeds and VIF computation) and details the number of models found and explored. If \texttt{n\_jobs} exceeds the allowed maximum, it is normalized and reported on the console.

\subsection{Output}
Models are exported to the specified CSV with normalized variable names, internal/external metrics, and CPU metadata. Error messages include the cause and exit with code 1.

\section{Operational good practices}
\begin{itemize}
  \item Review parameters before each run and save configuration files (\texttt{.conf}) for batch/CLI reproducibility.
  \item Adjust \texttt{n\_jobs} to available hardware to balance time and resources.
  \item Use \texttt{iterations\_mode = converge} to let EPR-S control iterations by convergence.
  \item Check collinearity with correlation and VIF thresholds; confirm robustness in \textbf{Validation} and \textbf{Diagnostics} before decisions.
  \item Keep the CSV clean and document exclusions; for external holdout, preserve identical headers and order.
\end{itemize}

\section{Troubleshooting and FAQ}
\begin{itemize}
  \item \textbf{The GUI does not open}: reinstall the corresponding build and ensure the binary is not blocked by antivirus.
  \item \textbf{CSV does not load}: verify path, permissions, and delimiter; use \emph{Preview data}.
  \item \textbf{No models exported}: review \emph{Minimum report R$^2$/Q$^2$} and \emph{VIF threshold}.
  \item \textbf{External validation fails}: the target column must match and the CSV must contain the same predictors.
  \item \textbf{Empty plot or missing labels}: select a valid model/dataset or load a results file.
  \item \textbf{Unexpected CPU usage}: check \texttt{n\_jobs}; the system normalizes it to at least 1 and reports adjustments.
\end{itemize}
