\section{Distribution Packages}
Packages are portable and do not require external Python installations or scientific libraries.

\subsection{Platform Support}
\begin{itemize}
  \item \textbf{Linux}: Single binary executable
  \item \textbf{macOS}: Application bundles (.app)
  \item \textbf{Windows}: Portable executables (.exe)
\end{itemize}

\subsection{Interface Variants}

\subsubsection{GUI-Only Versions}
\begin{itemize}
  \item Launch only the graphical user interface
  \item No terminal/console window displayed
  \item Available for: macOS (.app), Windows (.exe)
\end{itemize}

\subsubsection{GUI+CLI Versions}
\begin{itemize}
  \item Launch both graphical interface AND terminal/console
  \item Allow simultaneous GUI and command-line interaction
  \item Terminal remains open for CLI commands alongside GUI
  \item Available for: macOS (.app), Windows (.exe)
\end{itemize}

\subsubsection{Linux Behavior}
\begin{itemize}
  \item Single binary supports both modes
  \item GUI-only interface by default
  \item CLI accessible via command-line arguments
  \item No separate GUI+CLI variant needed
\end{itemize}

\medskip
CLI-enabled platforms allow running the full pipeline without the GUI by supplying a \texttt{config.conf} file.

\section{Requirements and installation}
The distributed packages are self-contained downloads and do not require installers or shortcut generators. Place each executable in a preferred folder. Optionally, add that folder to the system \texttt{PATH} so \verb|MLR-X| can launch from any directory (e.g., \verb|export PATH=/opt/MLRX:$PATH| on Linux or configuring environment variables on Windows). Choose the appropriate package and follow these steps:

\subsection{Setting up the portable packages}
\begin{itemize}
  \item \textbf{Linux}: download the binary, move it to a stable folder, and mark it as executable if needed (\texttt{chmod +x MLR-X}). To add it to your \texttt{PATH}, prepend the folder with \verb|export PATH=/path/to/folder:$PATH| or the equivalent shell profile setting. To register a desktop launcher:
        \begin{enumerate}[label=\arabic*.]
          \item Create the Linux executable (see the steps above), then copy the binary and the icon to a stable location, for example:
          \item \texttt{sudo mkdir -p /opt/MLRX}
          \item \texttt{sudo cp MLRX /opt/MLRX/}\\
                \texttt{sudo cp icon/icon.ico /opt/MLRX/}
          \item Update the \texttt{Exec} and \texttt{Icon} paths in \texttt{MLRX.desktop} if a different installation directory is used.
          \item Install the launcher by copying it to your applications directory and mark the file as executable:
          \item \texttt{chmod +x MLRX.desktop}
          \item \texttt{cp MLRX.desktop ~/.local/share/applications/}
          \item \# or for all users:\\
                \texttt{\# sudo cp MLRX.desktop /usr/share/applications/}
          \item Refresh the desktop launcher database (optional but recommended):
          \item \texttt{update-desktop-database ~/.local/share/applications}\\
                \texttt{\# or sudo update-desktop-database /usr/share/applications}
        \end{enumerate}
  \item \textbf{Windows GUI}: download the GUI package, place \texttt{MLR-X.exe} in a convenient folder, and run it directly.
  \item \textbf{Windows GUI+CLI}: same as Windows GUI, with the Run CLI option enabled.
  \item \textbf{macOS GUI}: download the app bundle and run it from your preferred location (e.g., move it to \texttt{/Applications} if desired).
  \item \textbf{macOS GUI+CLI}: same as macOS GUI, with the Run CLI option enabled.
\end{itemize}

\section{Interface (GUI) tour}
The main window presents a tabbed workflow.

\subsection{\textbf{Setup \& Run} tab}
Organizes dataset loading, splitting, model-search settings, and run controls. A banner at the top of this tab tracks the active task and its progress, while the lower status bar mirrors the current action. Tabs become available automatically once their prerequisites are filled in.
\paragraph{Dataset}
\begin{enumerate}[label=\arabic*.]
  \item Choose the CSV via the field or \emph{Browse} button.
  \item Set the delimiter (\verb|;| by default), target (dependent variable), and any non-predictor columns.
  \item Optionally exclude observations by ID or ranges (e.g., \texttt{3,9,4-8}).
  \item Enable \emph{Exclude nearly constant predictors} and provide the percentage threshold to drop near-zero-variance variables.
  \item Actions: \emph{Preview data}, \emph{Run analysis}, \emph{Cancel}. Status appears in the top-right corner; the active task shows in the lower-right footer.
\end{enumerate}

\paragraph{Data splitting}
Choose how to partition the dataset for training and testing.
\begin{itemize}
  \item \emph{None}: use the full dataset.
  \item \emph{Random}: specify \emph{Test size (\%)}.
  \item \emph{Manual}: provide train/test lists or ranges without overlap.
\end{itemize}
Selecting \emph{None} keeps external validation available later from the Validation tab by loading a separate file.

\paragraph{Settings}
  \begin{enumerate}[label=\arabic*.]
    \item Search method: \textbf{EPR-S} (Expand--Perturb--Reduce--Swap) or \textbf{All Subsets}. EPR-S is an original method in this software (doi.org/xxxxx).
    \item EPR-S parameters: \emph{Number of seeds}, \emph{Seed size}, and \emph{Set iterations/seed}. Seeds are independent starting points for exploring model combinations; recommendation buttons fill sensible defaults. Iterations per seed can run in \textit{auto} (estimate and apply a maximum), \textit{manual} (enter a maximum), or \textit{converge} (run until convergence; can be lengthy).
    \item Key parameters: \emph{Max predictors per model}, \emph{Significance level}, \emph{Correlation threshold}, \emph{VIF threshold}, \emph{Minimum report R$^2$}, \emph{Top models to report}, and \emph{Parallel jobs}. \emph{Top models to report} caps the best-scoring models exported to CSV, diagnostics tables, visualizations, and summaries (default 5000). Predictors correlation and Variance Inflation Factor (VIF) thresholds gate multicollinearity; Minimum report R$^2$ uses the training-set R$^2$ as a gatekeeper even when optimizing alternative metrics.
    \item Target metric and covariance: select \emph{R$^2$}, \emph{adj-R$^2$}, or \emph{Q$^2$} as the scoring objective and pick the covariance estimator (Non-robust, HC0--HC3). Non-robust keeps the standard OLS covariance (assumes homoskedasticity). HC0 replaces constant error variance with squared residuals (White, 1980). HC1 applies a small-sample degrees-of-freedom adjustment to HC0. HC2 down-weights high-leverage observations using $1/(1-h_i)$. HC3 uses $1/(1-h_i)^2$ for a jackknife-like correction, offering the strongest finite-sample protection against leverage.
    \item \emph{Clip predictions to}: set lower and/or upper bounds to constrain predicted values for scoring and exports.
  \end{enumerate}

\subsection{\textbf{Models} tab}
Displays three synchronized tables for training, internal validation, and external validation. The header controls allow loading a results file (\emph{Browse}, \emph{Load}, \emph{Clear}, \emph{Filter}), picking the sorting criterion, and filtering models by predictors.
\begin{itemize}
  \item \textbf{Top models training}: predictor set, predictor count (N pred), goodness-of-fit (R$^2$, adjusted R$^2$, RMSE, MAE, $s$), and multicollinearity (maximum/average VIF).
  \item \textbf{Top models internal validation (CV)}: LOO and k-fold metrics (Q$^2$, RMSE, MAE, $s$).
  \item \textbf{Top models external validation}: external prediction metrics Q$^2_{\mathrm{F1}}$, Q$^2_{\mathrm{F2}}$/R$^2_{\mathrm{ext}}$, Q$^2_{\mathrm{F3}}$, RMSE, and MAE.
\end{itemize}
All three tables update automatically and stay synchronized with the Validation, Diagnostics, Visualization, and Summary tabs once a model set exists (after a run or when loading saved results).

\subsection{\textbf{Validation} tab}
Configures cross-validation and external validation for the chosen models.
\begin{itemize}
  \item Model selection: \emph{Top}, \emph{Multiple} (range like \texttt{1-5,7,9}), or \emph{All}.
  \item \textbf{LOO and k-fold}: internal validation with \emph{Leave One Out (LOO)} and/or \emph{Folds} plus \emph{Repeats}.
  \item \textbf{External validation}: choose file and delimiter; requires the same target column as the training set.
\end{itemize}
Validation outputs append automatically to the Models tab when finished.

\subsection{\textbf{Diagnostics} tab}
Provides per-observation diagnostics and the correlation matrix for the active model.
\begin{itemize}
  \item Model selector with status and dataset filter (\emph{Training only}, \emph{Training and testing}, or \emph{Testing only}); holdout status propagates automatically.
  \item Observation diagnostics table with \emph{Observation}, \emph{Set}, \emph{Actual}, \emph{Predicted}, \emph{Predicted (LOO)}, \emph{Residual}, \emph{Residual (LOO)}, \emph{Z value}, \emph{Leverage}, \emph{Std. residual}, and \emph{Std. residual by LOO}.
  \item \emph{Export CSV} saves the currently visible diagnostics using the active dataset filter.
  \item Variable correlation matrix (lower triangle) with a toggle to include the dependent variable when available, plus CSV export.
\end{itemize}

\subsection{\textbf{Visualization} tab}
\begin{itemize}
  \item Model selector with availability status and dataset filter (\emph{Training only}, \emph{Training and testing}, or \emph{Testing only}); Y-randomization runs per model and can insert charts into the plot list when complete.
  \item Chart list: correlation heatmaps (with or without the dependent variable), Predicted vs Actual (standard and by LOO), residuals vs actual/predictions (standard and by LOO), Scale--Location (standard and by LOO), QQ residuals (standard and by LOO), residual distribution (standard and by LOO), Williams plot (standard and by LOO), Cook's distance vs leverage (standard and by LOO), leverage plot, and Predicted vs Leverage.
  \item Styling and overlays: legend toggle with four preset positions, labeling mode (observations or auto-detected extra column), identity line (Predicted vs Actual and QQ residuals), optional linear fit (Predicted vs Actual), $h^*$ threshold color for influence plots, grid toggle, and synchronized axes.
  \item Marker customization: separate color and marker-style pickers for training and testing points, plus marker size control; heatmaps disable incompatible controls automatically.
  \item Axes and export: manual X/Y limits, editable axis labels with font size, and high-resolution export of the current plot to PNG, TIFF, PDF, or SVG.
\end{itemize}

\subsection{\textbf{Summary} tab}
\begin{itemize}
  \item Model selector with status plus a \emph{Use recommended covariance} toggle (runs BP/White and leverage diagnostics before applying the suggested estimator) and \emph{Export Summary...} to save the full text dossier. A scrollable pane renders the current model summary.
  \item Overview: regression type, chosen covariance, search method, iterations per seed, target metric, models explored/found/reported (with threshold), significance level, observation counts, dependent variable, predictor list, k-fold plan, CPU times, clip-prediction bounds, and Y-randomization status.
  \item Covariance diagnostics (when the recommendation is enabled): recommended/applied type, Breusch--Pagan and White $p$-values, leverage threshold, maximum leverage, high-leverage count, and notes that triggered the robust choice.
  \item Coefficients and equation: coefficient table with standard errors, $t$, $p$, and confidence intervals; equations rendered with and without errors.
  \item Metrics: training block (R$^2$, adj-R$^2$, RMSE, MAE, $s$, VIF max/avg, $|r|_{\max}$), internal validation (LOO and k-fold with repeat note), and external validation (Q$^2_{\mathrm{F1}}$, Q$^2_{\mathrm{F2}}$, Q$^2_{\mathrm{F3}}$, RMSE, MAE).
\end{itemize}

\subsection{\textbf{Desktop} tab (Variable Explorer)}
\begin{itemize}
  \item Inputs: browse/load the CSV and an optional results file (unlocks model-aware options) with a readiness status line.
  \item Variable scope and axes: choose \emph{All variables} or \emph{Only variables of model} (enter an ID), then select X/Y axes. A \emph{Histogram} toggle switches to histogram mode with fixed-bin or custom-breakpoint controls.
  \item Plot controls: marker size/color/style, legend with selectable positions, gridlines, linear fit with optional R$^2$ display, partial residuals (when a model is active), aspect ratios (1, 4:3, 16:9, 9:16), manual axis limits and tick steps, and a dialog for axis labels and font size. Interactive labels can be added/dragged with left-click and cleared with right-click.
  \item Exports and filtered datasets: \emph{Save current plot} to PNG, TIFF, PDF, or SVG (high resolution). \emph{Create filtered dataset} creates a CSV containing observation/ID/non-variable columns plus predictors from the current model or a specified model range; the app warns about missing models in the requested range.
\end{itemize}

\subsection{\textbf{Help} tab}
Summarizes product information and support links. The About panel shows the MLR-X name, version, license notice, and tagline. Quick links open the homepage and contact email, and buttons provide the bundled manual, citation dialog, bug-report form, and donation page.

\section{Analysis flow and process handling}
\subsection{Before starting}
\begin{enumerate}
  \item Load and preview the dataset, confirming delimiter, target, exclusions, and split mode.
  \item Check that no other run is in progress, then choose where to save results when prompted (defaults to \texttt{models.csv}; a custom path is allowed).
\end{enumerate}

\subsection{Starting a run}
\begin{enumerate}
  \item Click \emph{Run analysis}. The app validates the configuration, reloads the dataset (including the split), and prepares the selected output file.
  \item The status switches to \emph{Running}, the \emph{Run analysis} button is disabled, \emph{Cancel} is enabled, and previous results, caches, and downstream tabs (Validation, Summary, Diagnostics, Visualization, Desktop) are reset.
  \item A run header plus ``Starting analysis...'' is appended to the log; the progress bar and step counter are reset for the search and post-processing stages.
\end{enumerate}

\subsection{Progress and cancellation}
\begin{itemize}
  \item The progress bar summarizes each stage, and the \emph{Log} lists model counts and timing as they occur.
  \item \emph{Cancel} stops workers, marks the status as \emph{Cancelling...}, and notes the request in the log. Errors set status to \emph{Error} and leave the run controls ready for another try.
\end{itemize}

\subsection{Completion and outputs}
\begin{enumerate}
  \item When finished, the status becomes \emph{Completed}; \emph{Run analysis} re-enables and \emph{Cancel} disables. Validation, Diagnostics, Visualization, and Summary tabs are unlocked with the new results, and any external holdout is refreshed if available.
  \item Results load automatically into the tables, summary pane, and downstream tabs, and the CSV is written to the confirmed path (\texttt{models.csv} by default). The header line embeds run metadata as JSON between \verb|#EPRS-S ... ;#|, and the most recent export path is remembered for the next session.
  \item If no models meet the reporting threshold, the app clears tables, keeps analysis tabs disabled, and shows an information dialog.
\end{enumerate}

\section{Search algorithms}
\subsection{EPR-S (Expand--Perturb--Reduce--Swap)}
Heuristic search that runs per seed using the target metric, predictor cap, correlation threshold, VIF veto, and reporting thresholds defined in Setup \& Run. Each seed follows a deterministic loop:
\begin{enumerate}
  \item Initialize with the seed's starting predictors and compute the baseline metric.
  \item \textbf{Expand}: add the single predictor that most improves the metric without violating the maximum predictors.
  \item \textbf{Perturb}: perform one-for-one swaps (drop one, add one) to escape local optima.
  \item \textbf{Reduce}: prune one predictor at a time when the metric stays the same or improves.
  \item If two active predictors exceed the correlation threshold, drop the one with weaker contribution (or lower target correlation when needed) and refresh the remaining set.
  \item \textbf{Swap recovery}: revisit previously dropped variables when swap logs suggest a possible gain.
  \item Stop when no further gains appear after repeated passes or when the iteration budget per seed is reached. Quality filters (VIF and minimum report R$^2$) keep only unique, eligible models sorted by the target metric.
\end{enumerate}
\medskip
EPR-S is an original method in this software (doi.org/xxxxx).

\subsection{All subsets (exhaustive)}
Enumerates every predictor combination up to \emph{Max predictors}, pre-filtering by the correlation threshold and applying VIF checks during evaluation. Progress tracking, cancellation, and the same export limits and reporting thresholds as EPR-S are enforced. Use this engine when the predictor space is small (practical below roughly 25 variables) because the number of combinations grows quickly.

\section{Result format and exports}
\subsection{Model CSV}
\begin{itemize}
  \item Semicolon-separated file with columns for model ID, predictors, predictor count, training metrics (R$^2$, adjusted R$^2$, RMSE, MAE, $s$), and multicollinearity indicators (maximum and average VIF).
  \item Internal validation columns append LOO and k-fold metrics (Q$^2$, RMSE, MAE, $s$); external validation columns append Q$^2_{\mathrm{F1}}$, Q$^2_{\mathrm{F2}}$ (R$^2_{\mathrm{ext}}$), Q$^2_{\mathrm{F3}}$, RMSE, $s$, and MAE.
  \item The end of the first row stores a serialized metadata block (dataset path, delimiter, dependent/ignored columns, constant-filter threshold, exclusions, target metric, search method, covariance type, iteration mode, split configuration, clipping limits, k-fold settings, validation file/delimiter, export limit, models found/explored/reported, and CPU times) so the UI can reload the configuration accurately.
  \item \emph{Export limit} caps the number of rows written; the exported table follows the training-order sort and mirrors the Models tab, including validation tables when available.
\end{itemize}

\subsection{Summary and reports}
\begin{itemize}
  \item Summary tab: \emph{Export Summary...} writes the currently selected model dossier (including the chosen covariance setting) to a text file.
  \item Diagnostics tab: \emph{Export CSV} saves the filtered observation diagnostics and the active correlation matrix (with or without the target) using semicolon-separated CSV.
  \item Visualization/plots: \emph{Save current plot} exports the visible chart to PNG, TIFF, PDF, or SVG with high-DPI output when needed.
\end{itemize}

\section{Command-line (CLI) usage}
Available on Linux and on the macOS and Windows GUI+CLI packages to run the full pipeline without launching the GUI. Provide exactly one argument: the path to a configuration file.

\subsection{Invocation}
\begin{lstlisting}[style=mlrx]
# Linux
./MLRX config.conf or ./MLRX path/to/config.conf
# macOS GUI+CLI
./MLRX config.conf or ./MLRX path/to/config.conf
# Windows GUI+CLI
MLRX.exe config.conf or MLRX.exe path/to/config.conf
\end{lstlisting}
If additional arguments are supplied, the program shows the expected syntax and exits.

\subsection{Configuration file structure}
Plain-text file with headed blocks (\# Dataset, \# Method, \# Validation, \# Output, etc.) and key--value pairs. List-type options allow selecting by index using \texttt{selected option} inside the block (for example, to pick a delimiter or target metric).
\subsection{Execution and progress}
The CLI loads the configuration, prints the dataset name, available predictors, search method, and (for EPR-S) the seed and iteration settings. It validates ranges (including manual iteration counts and split IDs), normalizes excessive parallel job requests, and streams progress updates for model searching and VIF calculations. Upon completion it reports models explored, models meeting the Minimum report R$^2$ threshold, filtered models kept after export limits, and CPU times.

\subsection{Output}
Results are exported to the specified CSV with semicolon separation, normalized variable names, training/internal/external metrics, and a metadata header so the GUI can reload the run. If no model satisfies the Minimum report R$^2$ filter, no CSV is written. Any parsing or runtime error is printed to stderr and exits with code 1.

\section{Operational good practices}
\begin{itemize}
  \item Validate inputs before running: preview the CSV to confirm the delimiter, target column, non-variable columns, and exclusions. Apply the constant filter when near-constant predictors should be removed. If planning an external holdout, ensure identical headers, column order, and target definition across all files.
  \item Choose the search strategy deliberately: use EPR-S for efficient exploration of large predictor spaces guided by stochastic seeds, and All Subsets when exhaustive coverage is required and longer runtimes are acceptable. Prefer EPR-S for large or high-dimensional designs; reserve All Subsets for small datasets or confirmatory analyses.
  \item Configure EPR-S parameters thoughtfully: set the \emph{Number of Seeds} to at least the number of predictors; use the \emph{Recommended} button for automatic calculation. Increase this value if premature convergence is observed—values above $\sim$1{,}000 seeds are recommended for robust searches. Choose a \emph{Seed size} close to half of the maximum number of predictors to balance exploration depth and computational feasibility.
  \item Leverage recommended defaults, then refine: allow the app to prefill EPR-S values using the recommendation buttons, then adjust \emph{Minimum report R$^2$} and the \emph{Export limit} (default 5,000) to control how many candidate models are retained and written to disk.
  \item Tune statistical thresholds before execution: define the \emph{Significance level} to control predictor inclusion. Set the \emph{Correlation threshold} (0--1) and \emph{Variance Inflation Factor (VIF) threshold}—preferably VIF $<$ 5—to mitigate multicollinearity. Revisit these thresholds if the filtering is too permissive or overly restrictive.
  \item Align parallelization with available hardware: set \emph{Parallel jobs} according to available CPU capacity. Although the app normalizes this value, requesting fewer jobs can reduce contention on shared or heavily loaded machines.
  \item Validate comprehensively and interpret jointly: always run internal validation (LOO or k-fold) and, when available, external validation. Review the \emph{Validation}, \emph{Diagnostics}, \emph{Visualization}, and \emph{Summary} tabs together to assess model stability and reliability. Enable the recommended covariance option in the Summary tab when statistical inference is required.
  \item Apply prediction clipping cautiously: use \emph{Clip predictions} only when the response variable has known physical or theoretical bounds, and ensure limits are consistent with the units and range of the training data.
  \item Preserve reproducibility: save all generated model CSVs and configuration files. These contain embedded metadata that allow future GUI or CLI sessions to reload and reproduce the exact run context.
\end{itemize}

\section{Parameter and chart reference}

\subsection{Key configuration parameters}
\begin{itemize}
  \item \textbf{Number of Seeds}: controls how many stochastic starting points EPR-S explores; higher counts broaden coverage and reduce convergence to suboptimal models.
  \item \textbf{Seed size}: sets the initial predictor count per seed; values near half the predictor limit balance exploration and computation.
  \item \textbf{Set iterations/seed}: caps or adapts the exploration depth per seed through \emph{auto}, manual, or convergence modes.
  \item \textbf{Max predictors per model}: upper bound on predictors allowed in any candidate model.
  \item \textbf{Significance level}: threshold for coefficient inclusion during model refinement.
  \item \textbf{Correlation threshold}: maximum tolerated pairwise correlation before pruning to control redundancy.
  \item \textbf{VIF threshold}: veto for models with multicollinearity above the specified variance inflation factor.
  \item \textbf{Minimum report R$^2$}: training R$^2$ cutoff applied before metrics are compared with the optimization target.
  \item \textbf{Top models to report}: limit on retained models; aligned with the CSV export limit.
  \item \textbf{Target metric}: optimization objective (R$^2$, adjusted R$^2$, or Q$^2$).
  \item \textbf{Covariance estimator}: robust or non-robust variance choice for standard errors and inference.
  \item \textbf{Clip predictions}: lower/upper bounds used when predictions must respect known limits.
  \item \textbf{Parallel jobs}: requested worker count for parallel tasks, normalized to available CPU cores.
\end{itemize}

\subsection{Interpreting key charts}
\begin{itemize}
  \item \textbf{Correlation heatmaps}: visualize pairwise predictor relationships; dark blocks expose multicollinearity risks.
  \item \textbf{Predicted vs Actual}: alignment with the identity line indicates accurate calibration; systematic deviations flag bias.
  \item \textbf{Residuals vs Predicted/Actual}: random scatter suggests homoskedasticity and linearity; structure signals misspecification.
  \item \textbf{Scale--Location}: spreads residuals against fitted values to reveal variance patterns; flattening curves indicate stable variance.
  \item \textbf{Residual distribution}: histogram or density of residuals to spot asymmetry, heavy tails, or outliers relative to a centered, bell-shaped pattern.
  \item \textbf{QQ residuals}: departures from the diagonal highlight non-normal residual distributions.
  \item \textbf{Leverage plot}: leverage values across observations reveal influential points relative to the threshold.
  \item \textbf{Cook's distance vs leverage}: high distances at elevated leverage identify observations with strong influence on estimates.
  \item \textbf{Williams plot}: combines leverage and standardized residuals to spot influential observations beyond leverage and residual cutoffs.
  \item \textbf{Predicted vs Leverage}: blends fitted values with leverage to isolate influence patterns across the prediction range.
\end{itemize}

\section{Troubleshooting and FAQ}
\begin{itemize}
  \item \textbf{The GUI does not open}: download the corresponding package again and ensure the binary is not blocked by antivirus.
  \item \textbf{CSV does not load}: verify path, permissions, and delimiter; use \emph{Preview data}.
  \item \textbf{No models exported}: review \emph{Minimum report R$^2$} and \emph{VIF threshold}.
  \item \textbf{External validation fails}: the target column must match and the CSV must contain the same predictors.
  \item \textbf{Empty plot or missing labels}: select a valid model/dataset or load a results file.
  \item \textbf{Unexpected CPU usage}: check \emph{Parallel jobs}; the system normalizes the value to at least 1 and reports adjustments.
\end{itemize}
Report any persisting issues or bugs at \url{https://github.com/Jacksonalcazar/MLR-X/issues} so they can be addressed promptly.
